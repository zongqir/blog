# 背景

当前博客已经具备基础的信息架构：

- 首页有精选、热门、专题连载和文章列表
- 文章页有摘要、标签、阅读时长和自动目录
- 归档、标签、关于等关键页面都已存在

但整体视觉语言仍然偏传统技术博客：

1. 首页更像功能列表，而不是有明确判断和策展感的刊物入口。
2. 文章页已经具备阅读辅助能力，但排版和版面节奏还不够“出版化”。
3. 整站风格虽有暗色基调，但更接近常见深色主题，而不是“编辑型技术杂志”。

这次改造的目标不是重写内容，也不是给博客加更多功能，而是在不改动文章正文立场和主体内容的前提下，重做博客外壳与阅读体验。

# 目标

本次重设计要同时满足以下约束：

1. 首页建立“有判断的技术刊物”气质，而不是个人自我介绍页。
2. 不强迫作者书写“我是谁”，避免把首页做成个人展示卡。
3. 阅读体验优先，特别是长文阅读、标题层级、正文节奏和继续阅读路径。
4. 保留现有内容资产与信息组织能力，不大规模重写 frontmatter 或文章正文。
5. 视觉上形成明确识别度，但不过度炫技，不损害可读性。

# 设计结论

最终采用：

- 方向：编辑型技术杂志感
- 首页定位：议题主编型，而非作者介绍型
- 文章页定位：长文优先、刊物式阅读体验

不采用以下方向：

- 不做强调个人履历和头像的个人品牌首页
- 不做偏后台面板感的知识库首页
- 不做强插画、强装饰、维护成本过高的封面站

# 首页信息架构

首页采用以下阅读顺序：

1. 主编导语区
2. 本期重点文章
3. 专题连载与近期文章
4. 站点状态类辅助信息

## 1. 主编导语区

第一屏不写“宗旗是谁”，而是直接回答：

- 这里长期讨论什么问题
- 这些文章帮助读者看清什么取舍

结构采用双区：

- 左侧为强观点主标题与 2 到 3 句导语
- 右侧为“当前关注议题”列表，承接站点长期母题

议题建议围绕：

- SaaS 化
- 架构演进
- 稳定性治理
- 性能优化

## 2. 本期重点文章

保留现有精选文章逻辑，但表达方式升级为“封面导读”：

- 文章标题为主视觉焦点
- 元信息压低权重
- 描述和标签形成继续阅读入口

这个区块的角色不是“推荐卡片”，而是当前站点最值得先读的一篇文章。

## 3. 专题连载与近期文章

专题连载用于承接“体系化阅读”，近期文章用于承接“持续更新”。

两者都保留，但视觉上应明显区别：

- 专题更像栏目入口
- 近期文章更像期刊目录

## 4. 辅助信息

站点活跃、内容规模、连载数量等状态保留，但降级为低权重辅助信号。

它们用于增加真实更新感，不参与第一屏叙事中心。

# 文章页设计

文章页采用“刊首导语 + 稳定正文列 + 轻量辅助边栏”的阅读模型。

## 1. 刊首导语

文章头部保留现有信息，但重新组织为更清晰的阅读入口：

- 标题
- 摘要
- 发布时间 / 更新时间
- 阅读时长
- 标签

目标是让读者在很短时间内完成“这篇文章是否值得投入阅读”的判断。

## 2. 正文排版

正文排版是本次改造的核心。

需要重点优化：

- 正文列宽
- 标题层级
- 段落留白
- 列表、引用、代码块和表格样式
- 标题与段落之间的节奏关系

目标不是做成文档站，而是让博客长文读起来更像经过编辑处理的出版页面。

## 3. 目录

保留现有自动目录能力，延续右侧固定、移动端折叠的方案。

目录只作为阅读辅助，不作为视觉中心：

- 桌面端弱化工具感
- 移动端默认收起
- 当前项高亮表达保持轻量

## 4. 继续阅读

相关文章区块改造为“继续阅读”，强调顺滑的阅读延伸，而不是标准化推荐模块。

优先延续专题和相近议题的阅读路径。

# 视觉语言

## 1. 总体气质

整体视觉采用“夜间编辑部”而不是“深色程序员主题”。

特征如下：

- 深色底，但不是纯黑
- 含墨蓝、烟灰、暖纸张感的综合色温
- 保留理性气质，但弱化工具界面感

## 2. 排版语气

排版比装饰更重要。

核心做法：

- 首页主标题具备更强张力
- 正文标题层级清晰
- 辅助文字和元信息刻意降噪
- 卡片弱组件化，强版面化

## 3. 组件表达

不采用典型 SaaS 面板视觉。

将以以下方式替代：

- 刊物式分隔线
- 更轻的卡片边界
- 更像导读条目和栏目块的列表样式
- 更克制的 hover 和滚动反馈

## 4. 动效原则

只做少量高价值动效：

- 首屏分层出现
- 卡片轻微位移或高亮
- 目录和阅读进度的安静反馈

禁止使用炫技型大动效，避免破坏阅读专注。

# 范围边界

本次改造只处理博客壳层与阅读体验层，不改动内容立场。

## 会修改的内容

- 首页模板结构
- 默认布局的导航和页脚
- 文章页头部和相关文章区域结构
- 全站样式系统
- 少量与阅读体验直接相关的脚本
- 归档、标签、关于等页面的展示层

## 不修改的内容

- 已有文章正文内容
- 作者观点和论证
- 文章 frontmatter 语义
- 站点构建方式
- 标签和专题的核心数据来源

# 验收标准

本次设计完成的标准如下：

1. 首页第一屏不出现作者自我介绍，但能清晰建立站点判断气质。
2. 首页能形成“导语 -> 重点文章 -> 专题/近期文章”的清晰阅读路径。
3. 文章页正文阅读体验明显提升，尤其是长文排版、代码块和引用样式。
4. 目录继续可用，但不会抢正文入口。
5. 整站风格与当前内容调性一致，不会让博客看起来像另一种人格。
6. 归档、标签、关于等页面在新视觉体系下保持统一。
7. Jekyll 构建通过，关键页面可正常渲染。

# 实施建议

实现阶段建议按以下顺序推进：

1. 先重构全站视觉 token 和基础排版体系
2. 再重做首页第一屏与内容区块
3. 接着优化文章页头部、正文和继续阅读区
4. 最后统一归档、标签、关于等次级页面的壳层表达

这样可以在不触碰文章内容的前提下，最大化提升阅读体验与整站辨识度。
