背景

当前文章页已经有自动生成的目录，但目录仍然位于正文之前。

这会带来两个直接问题：

  1. 目录较长时，首屏阅读路径会变成“标题 -> 长目录 -> 正文”，正文被明显后置。
  2. 目录虽然存在，但它的占位方式仍然像正文的一部分，而不是阅读过程中的辅助导航。

这次改造的目标不是“再做一个更显眼的目录”，而是重新定义目录在文章页中的角色：

  • 正文是主任务。
  • 目录是持续可用、但不抢入口的阅读辅助。

设计结论

最终采用：

  • 桌面端右侧固定目录
  • 移动端默认折叠目录

不采用左侧目录，也不采用桌面端悬浮按钮抽屉方案。

为什么采用这个方案

1. 先解决正文入口被目录挤压的问题

用户当前最直接的不适不是“目录不好用”，而是“看了半天目录，才开始正文”。

因此优先级最高的是让正文重新回到首屏主路径，而不是继续强化顶部目录。

2. 右侧比左侧更符合博客阅读

左侧目录更像文档站、知识库或后台系统，结构感更强，但也更容易先抢视线。

博客文章的主路径仍然是连续阅读。目录应当让用户在需要时快速定位，而不是在开始阅读前强行要求先理解结构。

右侧固定目录更适合作为“扫一眼就能知道读到哪里”的辅助层。

3. 移动端必须默认收起

移动端横向空间不足，任何完整目录的默认展开都会再次复现“正文被往下推”的问题。

因此移动端目录只保留入口，不默认展开。用户需要时再打开，点击目录项后自动收起。

信息架构

文章页的阅读结构调整为：

  1. 标题区
  2. 正文主栏
  3. 右侧目录辅助栏（仅桌面端显示）
  4. 相关文章

移动端调整为：

  1. 标题区
  2. 折叠目录入口
  3. 正文
  4. 相关文章

目录不再作为正文前的一整块内容插入阅读流中。

桌面端设计

布局

文章页采用“正文主栏 + 右侧辅助栏”的双栏结构，但必须保持主次明确：

  • 正文保持当前偏窄的阅读宽度，不追求铺满宽屏。
  • 目录栏为较窄辅助区，只承担导航职责。
  • 整体视觉中心仍在正文，不在目录。

目录栏只在足够宽的视口中出现。窄屏平板或小笔记本不强行保留双栏。

目录行为

目录栏使用 sticky 固定：

  • 当用户开始阅读正文后，目录栏保持在视口内。
  • 目录最大高度不超过视口可用高度。
  • 如果目录过长，则目录区域内部滚动，而不是让整页因为目录过高失衡。

目录项范围继续限制为：

  • h2
  • h3

不纳入更深层级，避免目录本身再次变成阅读负担。

视觉权重

目录视觉上要明显轻于正文：

  • 小于正文的字号
  • 更弱的色彩对比
  • 当前项使用细边线或轻量高亮
  • 不使用大面积高饱和背景
  • 不做厚重卡片感

目标是“可见，但不抢读”。

当前章节高亮

保留当前章节高亮能力，但表达方式收敛：

  • 当前目录项文字略加强调
  • 左侧保留一条细线作为进度提示
  • 非当前项维持低对比状态

这样既能帮助定位，又不会让目录变成第二阅读中心。

移动端设计

默认状态

移动端目录默认收起。

标题区之后、正文之前提供一个单独的“目录”入口。入口本身应当轻量，不做高权重宣传式按钮。

展开方式

点击入口后，在当前位置下方展开目录面板：

  • 展开区域包含完整目录
  • 再次点击同一入口可收起
  • 点击目录项后跳转到对应标题，并自动收起面板

这样既保留导航能力，又不强迫用户在进入正文前先扫完整个大纲。

动效

移动端可以保留轻量反馈：

  • 箭头旋转
  • 面板短时展开/收起过渡

但动效应当足够短，且遵守减少动态效果的无障碍偏好。

显示规则

桌面端显示规则

满足以下条件时显示右侧目录：

  1. 视口宽度达到桌面断点
  2. 正文中存在足够数量的 h2 或 h3

如果文章非常短、标题层级过少，则不显示侧边目录，避免为目录而目录。

移动端显示规则

满足以下条件时显示折叠目录入口：

  1. 正文存在目录项
  2. 当前为移动端断点

如果目录项数量很少，也可以直接隐藏入口，避免增加无意义控件。

兼容性与异常情况

超长目录

当文章层级很多时：

  • 桌面端目录区域内部滚动
  • 移动端目录面板可自然纵向展开，但仍应保持整体可读

没有目录项的文章

如果正文没有 h2 或 h3：

  • 桌面端不渲染目录栏
  • 移动端不显示目录入口

锚点跳转

点击目录项后，页面应平滑跳转到对应标题，并继续保持当前项高亮正确。

如果未来引入固定头部偏移，需要同时校正标题锚点的可见位置。

实现边界

这次改造只处理文章页目录的布局与交互，不扩展到以下方向：

  • 不重做整站视觉风格
  • 不改文章正文排版体系
  • 不新增更深层级目录生成规则
  • 不把目录做成独立浮动抽屉系统
  • 不把左侧目录作为备选同时上线

目标是先把文章页阅读路径修正到合理状态，再决定是否做进一步强化。

验收标准

本次设计的完成标准如下：

  1. 桌面端打开长文章时，标题后能直接进入正文，不再先看到整块长目录。
  2. 桌面端目录始终位于正文右侧，随滚动保持可用，并正确高亮当前章节。
  3. 超长目录不会把页面整体撑坏，而是在目录区域内部滚动。
  4. 移动端目录默认收起，只保留清晰但不突兀的入口。
  5. 移动端点击目录项后能跳转并自动收起，不影响继续阅读。
  6. 短文章或无层级文章不会强行显示目录控件。
  7. 保留现有自动生成目录和当前章节高亮能力，不引入行为回退。

测试与验证建议

实现阶段至少验证以下场景：

  1. 长文章桌面端阅读路径是否顺畅。
  2. 短文章是否正确隐藏目录。
  3. 目录项很多时右侧栏是否可滚动且不遮挡内容。
  4. 手机宽度下目录是否默认收起。
  5. 点击目录项后的跳转、收起和高亮状态是否正确。
  6. 缩放窗口跨越断点时，目录布局是否正确切换。
  7. prefers-reduced-motion 下是否不会出现多余动画。

后续实施建议

实现时优先复用现有目录生成与高亮脚本，只调整：

  • 文章页 DOM 结构
  • 桌面端布局样式
  • 移动端折叠交互

这样可以在最小改动范围内完成体验升级，避免为了目录改造牵动整站主题结构。
